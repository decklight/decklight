Built to Be Written by Agents

Authored by AI agents and humans alike — that's the whole idea

  • A deck is one HTML file — no build, no bundler, no framework
  • Describe a slide in plain English; your coding agent writes the Decklight
  • decklight init drops a skill so the agent knows the real contract, not a guess
  • Every feature is verifiable by a headless render — so an agent can check its own work without eyes

Here's what makes Decklight different, and it's the through-line for everything else: it was designed from the start to be authored by agents as much as by people.

Truthful Terminals

The Authoring Loop

Scaffold, live-edit with an agent in the room, ship one file

  • decklight init — scaffolds a deck and an agent skill (.claude/skills/ + AGENTS.md)
  • decklight dev — live reload; edit the file in your editor or the browser
  • A asks an installed agent — Claude Code, Codex, Gemini, Copilot, Aider… — to edit the deck; the browser reloads when it saves
  • E notes · L layouts · Z undo — every edit on one history, auto-committed
  • decklight bundle — flatten to one offline HTML file to hand off

So what does authoring actually feel like? Five moves.

⟨CLICK⟩

decklight init scaffolds a starter deck, and — this is the agent-native part — it also writes a skill file, sliced from the spec, so any agent in the repo knows exactly how Decklight works.

⟨CLICK⟩

decklight dev serves the deck with live reload. Edit it in your own editor, or right in the browser.

⟨CLICK⟩

And here's the loop that makes it sing: press A, type what you want changed, and an installed coding agent — Claude Code, Codex, Gemini, whatever's on your PATH — edits the file headlessly. The browser reloads the moment it saves.

⟨CLICK⟩

E edits notes, L cycles layouts, Z undoes any of it — including the agent's edits — all on one history, auto-committed as you go.

⟨CLICK⟩

When it's ready, decklight bundle flattens everything into a single offline file you can email.

One File to Ship

Runtime, themes, casts and narration — inlined

decklight bundle deck.html --themes all   # self-contained, offline
decklight bundle deck.html --all          # merge a whole playlist

Shipping is one command. bundle inlines the runtime, the themes you chose, the terminal casts, even the narration —

⟨CLICK⟩

— into a single HTML file. Email it, host it, open it on a plane. And with dash-dash-all it can merge an entire playlist of decks into one file with in-file navigation. This whole tour ships exactly this way.
